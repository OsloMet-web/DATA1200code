Welcome to DATA1200 Inclusive web-design

The pages you find here are coded to show specific examples. I have attempted to minimise the code and removed anything that could create clutter and make the educational content less obvious. Therefore, these pages are not pretty and they may lack some accessibility best-practice.

How to use these files

You should read both the text on the page and the <!--comments--> inside the HTML and CSS files (/* comments */). Sometimes, the page may look bad because I have kept the code too simplistic, but sometimes I have "commented out" code — i.e. used a comment syntax to hide code from the browser. I have attempted to add some helpful comments as well. But as this has been done in relation to specific topics I intend to cover in class, these comments are not exhaustive.

Sample pages

I have created a few pages. First of all, there are some super-simple HTML-pages listed below. These are used in class just like examples of how simple it can be. But I could not help it, with a nod to a much-used but now irrelevant, early repository of code, I have included a bare-minimum 'Zen garden' to be used with computers who use dark mode, or who switch modes (you should try!). Hopefully, I shall have time to show you in class.

  • The absolutely least code you can make a browser render: 'simplest.html'
  • Slightly more code, and enough meta-code (in the head, and with DOCTYPE) to make it "standard compliant" HTML5: 'simplest-legal.html'
  • 'simple-data1200.html' use special characters, and therefore needs the meta-code <meta charset="utf-8">; try to disable the code and see how different browsers solves this
  • An image is added in 'future-cooler.html'; you should look at the links in that page for more information about including images
  • (On the topic of images: I suggest you check out "Hero image" as a design element)

As a "site", a collection of interconnected pages with a uniform design, the Flower Farm leaves much to be desired! But there is a method to this madness. The first page is rather simple, but please play around with margin (or padding). The next – 'Farm' — has one static image (<img>) and an example of why you may use the <picture>-element.

Then follows an example of scaleability in web pages. While the page you are reading now uses a flex-box (like the footer in the Flower farm-site), the Flowers-page uses grid lay-out.

Then there are pages concerning boxes — visible and invisible. (Will be covered in a lecture.) And lastly, "About" was intended to be about scaling. And whatever else I might need to showcase in class.
